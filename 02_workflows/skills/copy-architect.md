# Skill: Copy Architect

## Trigger
When writing or rewriting any user-facing text: CTAs, page headlines, paywall messages, error states, onboarding flows, notifications, or email copy.

## Outcome
3–5 copy variants for A/B testing, each with tone notes and guidance on which audience segment it targets.

## Context Files
Read first:
- `01_context/product/4_COPY_SYSTEM.md` - Voice (authoritative + accessible), tone (warm, precise, never vague)
- `01_context/product/3_USER_FLOWS.md` - Personas (skeptics, believers, curious, astrology-aware)
- `01_context/product/7_SUCCESS_METRICS.md` - Conversion targets for messaging

## Steps
1. **Identify the surface**: Page headline, button label, error message, onboarding step, notification, email subject, etc.
2. **Read COPY_SYSTEM.md**: Confirm Vedic authority tone; ensure no mystical hand-waving (e.g., ✓ "Vedically-computed" not ✗ "cosmic energy")
3. **Understand audience context**: Check USER_SEGMENTS.md; which persona is this copy targeting?
4. **Identify the micro-goal**: What should the user do? (Sign up, upgrade, trust the result, retry after error, etc.)
5. **Write 3–5 variants**:
   - **V1 (Authoritative)**: Lead with Vedic credibility and precision; for astrology-aware users
   - **V2 (Curious-friendly)**: Explain the "why" without jargon; for skeptics entering funnel
   - **V3 (Urgency)**: Scarcity/FOMO angle if relevant (e.g., limited spots in cohort)
   - **V4 (Benefit-forward)**: Focus on outcome user cares about (clarity, career insight, etc.)
   - **V5 (Social proof)**: Mention user count or testimonial if available
6. **Tone check**: Ensure each variant matches COPY_SYSTEM.md—warm, precise, never vague
7. **Length check**: CTAs ≤5 words, headlines ≤12 words (mobile-friendly)
8. **Test for clarity**: Reread each variant; if any phrase feels mystical or imprecise, rewrite
9. **Output A/B test brief**: Document which segment, micro-goal, and tracking metric for each variant

## Quality Gates
- All variants must pass "no vague mysticism" test (cite COPY_SYSTEM.md)
- At least one variant must be tested on each user segment (A/B test plan included)
- Copy must be ≤80 chars for mobile CTAs, ≤160 chars for headlines
- No variant can use words like "cosmic energy", "mystical", "fate", "destiny" without scientific context

## Edge Cases
- **Error messages**: Variants should be reassuring + actionable (never accusatory)
- **Paywall messaging**: Must build value before asking for payment; reference pricing tiers in brief
- **Onboarding copy**: Varies by entry point (organic, ad, referral); create variants for each
- **Notification copy**: Keep to ≤100 chars; include action button label
